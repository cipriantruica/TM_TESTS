From: hamilton@romulus.ehs.uiuc.edu (Wayne Hamilton) Subject: Re: Null modem: 25 pin serial  Bruce Maynard (drac@uumeme.chi.il.us) wrote: > In article  Mitch Lewars  writes: > >Can someone give me the specs > >on a null modem cable, I ferget > >which get crossed.... =-) > >    mal112@psuvm.psu.edu  > that is an easy one... carry all 25 pins straight thru, with the exception of > pins 2 & 3, which should be crossed...  as someone else already mentioned, do not  carry thru  the other 23 pins.  plan A, minimal null modem: \tSG(7)   -  SG(7) \tTD(2)   -  RD(3) \tRD(3)   -  TD(2) if you are transfering files betwen 2 PCs, this ought to work (unless you have a persnickity comm program).  plan B, null modem with modem control: \tSG(7)   -  SG(7) \tTD(2)   -  RD(3) \tRD(3)   -  TD(2) \tDCD(8)  -  DTR(20) \tDTR(20) -  DCD(8) some comm software will wait for DCD before transferring anything. such software will raise DTR to enable the modem to accept a call. with this cable, each side is DTR will drive the other is DCD.  plan C, null modem with hardware flow control: \tSG(7)   -  SG(7) \tTD(2)   -  RD(3) \tRD(3)   -  TD(2) \tRTS(4)  -  CTS(5) \tCTS(5)  -  RTS(4) for high-speed transfers, you want the flow control.  plan D, deluxe null modem (combine B and C): \tSG(7)   -  SG(7) \tTD(2)   -  RD(3) \tRD(3)   -  TD(2) \tRTS(4)  -  CTS(5) \tCTS(5)  -  RTS(4) \tDCD(8)  -  DTR(20) \tDTR(20) -  DCD(8) this one is ideal.  it leaves out DSR and RI (rarely used anymore). if you are really paranoid, or you just have an 8th wire to spend, you might add: \tFG(1)   -  FG(1)  the pin numbers above are (obviously) for 25-pin connectors. i do not have a 9-pin pinout handy.  -- \twayne hamilton I\'net:\thamilton@osiris.cso.uiuc.edu Lowtek:\tBox 476, Urbana, IL 61801; (217)384-4310(voice), -4311(BBS)
